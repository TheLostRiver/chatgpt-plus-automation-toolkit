# Security Policy

## 支持范围

默认支持 `main` 分支最新代码。

## 报告安全问题

如果你发现以下问题，请不要在公开 Issue 直接贴敏感细节：

- 凭据泄露风险（API key、token、账号信息）
- 远程命令执行/任意文件写入
- 认证绕过、权限越权
- 敏感数据明文落盘

建议通过私密方式联系维护者，并提供：

1. 影响范围
2. 复现步骤
3. 风险评估
4. 临时缓解方案（如有）

## 安全基线

- 不在仓库提交真实密钥和个人数据
- 使用 `.env.example` 提供配置模板
- 运行产物与缓存目录通过 `.gitignore` 隔离
